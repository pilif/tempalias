What we do

We store the information you provide when creating the alias so we can redirect mail sent to that alias and expire it as needed.

We use the Google Analytics service to get an idea how the website is being used and how much it's accessed.

All mail sent to us will be forwarded to a smart host which will log the target address but no other information contained in the alias (it can't - that information was lost already when it sees the email).

We will store the settings you last used to create an alias in a cookie, so we can preset those when you visit our site again.

We cannot delete an expired alias the moment it expires, but it's guaranteed to be impossible for an email to be delivered to an expired alias - even under highly concurrent stress.

We prune expired aliases every 30 minutes. This process is irreversible.

We keep a list of every alias ever generated as to make sure that nobody else gets an alias that was previously generated for you. All other information stored with the alias (besides its name) is deleted on expiration though.

What we don't do

We don't log the emails content.

We don't log alias resolutions and their state.

We will never use the target address you provided for anything other than to relay mail to it within the validity settings you have chosen. We will never send any email out that is not in the context of an alias resolution.

We don't associate any information gathered by Google Analytics with any email alias the website visit generated.

We hope that these statements clearly show our intentions. If you have a concern about something we have forgotten to list above, please contact us at pilif@gnegg.ch. If you are capable of reading JavaScript code, you can have a look at the code that's running tempalias.com on http://github.com/pilif/tempalias.
